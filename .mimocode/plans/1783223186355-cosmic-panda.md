# Netlify Identity Bug修复计划

## 问题分析

根据研究，发现两个主要问题：

### 问题1: 退出登录按钮无法正常工作
**原因**: `netlifyIdentity.logout()` 不会清除 `nf_jwt` cookie（GitHub Issue #502）
- localStorage 中的 `gotrue.user` 会被清除
- 但 `nf_jwt` cookie 仍然存在
- 用户可以继续访问受保护的页面

### 问题2: 邮箱验证后再次登录仍显示"Email not confirmed"
**原因**: 验证流程可能没有正确完成
- 用户点击确认邮件后被重定向到网站
- 但验证状态可能没有正确更新
- 需要确保在验证流程中正确处理回调

## 修复方案

### 修复1: 手动清除 cookie

在 `logout()` 函数中添加手动清除 cookie 的代码：

```javascript
function logout() {
  // 清除 nf_jwt cookie
  document.cookie = 'nf_jwt=; path=/; expires=Thu, 01 Jan 1970 00:00:00 GMT';
  
  // 调用 Netlify Identity 的 logout
  netlifyIdentity.logout();
  
  // 跳转到登录页
  window.location.href = '/login.html';
}
```

### 修复2: 改进验证流程处理

确保在验证流程中正确处理回调：

```javascript
// 检查是否是验证/邀请/恢复流程（hash fragment）
const hash = window.location.hash;
const isVerificationFlow = hash.includes('confirmation_token') || 
                          hash.includes('invite_token') || 
                          hash.includes('recovery_token');

// 如果是验证流程，等待验证完成
if (isVerificationFlow) {
  netlifyIdentity.on('init', user => {
    if (user) {
      // 验证成功，跳转到主页
      window.location.href = '/';
    }
  });
}
```

## 修改文件

1. **js/auth.js** - 主要修改文件
   - 改进 `logout()` 函数，手动清除 cookie
   - 改进验证流程处理逻辑

2. **index.html** - 可能需要添加验证状态显示

## 验证步骤

1. 测试退出登录功能：
   - 点击退出登录按钮
   - 确认跳转到登录页
   - 尝试直接访问主页，确认被重定向

2. 测试邮箱验证流程：
   - 注册新用户
   - 收到确认邮件
   - 点击确认链接
   - 确认可以正常登录

3. 测试邀请流程：
   - 在 Netlify 后台邀请用户
   - 用户点击邀请链接
   - 设置密码并登录
