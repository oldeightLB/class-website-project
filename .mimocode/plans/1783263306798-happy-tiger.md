# Netlify Identity 验证流程修复计划 v3

## 问题根因

通过源码分析，`store.verifyToken` 处理流程：
```
1. Widget 检测到 hash fragment 中的 confirmation_token
2. store.modal.isOpen = true   → 打开模态框
3. gotrue.confirm(token)       → 验证邮箱
4. store.user = user           → 验证成功，设置用户
5. reaction 触发 login 事件   → 我们的代码立即跳转主页
6. 页面跳转，模态框消失       → 用户只看到一闪而过
```

**这就是用户描述的现象**："跳了一会netlify的页面，但只是一会就渐隐了"

## 修复方案

**核心思路**：在验证流程中延迟跳转，让 Widget 有时间显示验证结果。

### 修改 login.html

```html
<script>
  netlifyIdentity.init();

  // 检查是否是验证流程
  const hash = window.location.hash;
  const isVerificationFlow = hash.includes('confirmation_token') || 
                            hash.includes('invite_token') || 
                            hash.includes('recovery_token');

  // 登录成功事件
  netlifyIdentity.on('login', user => {
    // 验证流程中延迟跳转
    if (isVerificationFlow) {
      history.replaceState(null, '', window.location.pathname);
      setTimeout(() => {
        window.location.href = '/';
      }, 1500);
    } else {
      window.location.href = '/';
    }
  });

  // 错误事件
  netlifyIdentity.on('error', err => {
    console.error('Netlify Identity error:', err);
  });

  // 打开登录模态框
  document.getElementById('login-btn').addEventListener('click', () => {
    netlifyIdentity.open('login');
  });

  // 打开注册模态框
  document.getElementById('signup-btn').addEventListener('click', () => {
    netlifyIdentity.open('signup');
  });
</script>
```

### auth.js 不变

### index.html 不变

## 关键改动

1. **验证流程延迟跳转** - 1500ms 后跳转，让用户看到 Widget 的验证成功界面
2. **移除 alert** - 避免干扰用户体验
3. **保持代码简洁** - 让 Widget 自己管理模态框

## 关于"没有弹窗登录对话框"

用户需要点击"登录"按钮才会打开 Widget 的登录模态框。这是正常行为。

## 关于"没有退出登录选项"

登录页不显示退出按钮（用户还未登录），这是正确的。

## 如果验证仍然失败

在 Netlify 后台手动确认用户：
1. Identity → Users
2. 用户旁边的 "..." → "Confirm"
