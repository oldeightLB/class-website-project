# Plan: Fix Netlify Identity Auth Bugs + Redesign Login Page

## Problem Summary

Three issues:
1. **Bug 1 — Overlay blocks page**: Netlify Identity widget modal stays open, covering all content
2. **Bug 2 — Email confirmation fails**: Clicking confirmation link doesn't verify the email
3. **Design — Login page**: Show header/title, not fullscreen; reveal full content after auth

## Root Cause Analysis

Netlify's official docs state:
> "Without this call [handleAuthCallback()], confirmation links and OAuth redirects won't complete the authentication flow"

**The two bugs share one root cause**: The `netlify-identity-widget` detects `#confirmation_token=xxx` in the URL hash and auto-opens its modal to process the token. But the confirmation doesn't complete → modal stays open = overlay blocking page.

The widget README claims "Simply include the widget on your site, and things like invites, confirmation codes, etc, will start working" but this is unreliable.

## Solution: Keep widget CDN, add explicit confirmation handling

Since the user requires pure static HTML (no npm), we keep the `netlify-identity-widget` CDN. The fix:

1. **Call `netlifyIdentity.init()` FIRST** — before any event listeners
2. **Do NOT redirect while hash contains verification tokens** — let widget process them
3. **Listen for events to handle outcomes**:
   - `login` event → confirmation succeeded → close modal, clear hash, redirect
   - `error` event → confirmation failed → close modal, show error
4. **Manual confirmation fallback**: If widget fails, use `netlifyIdentity.gotrue.confirm(token)` directly
5. **Timeout fallback**: Close modal after 3s if still open

## Implementation

### 1. `js/auth.js` — Rewrite auth logic

```js
// On pages other than login.html, if hash has verification tokens,
// redirect to login.html with the hash preserved
if (isVerificationFlow && currentPage !== 'login.html') {
  window.location.href = '/login.html' + hash;
  return;
}

// On login.html, let widget process verification tokens
if (isVerificationFlow) {
  netlifyIdentity.init();
  
  netlifyIdentity.on('login', user => {
    netlifyIdentity.close();
    history.replaceState(null, '', window.location.pathname);
    window.location.href = '/';
  });
  
  netlifyIdentity.on('error', err => {
    console.error('Verification error:', err);
    netlifyIdentity.close();
    history.replaceState(null, '', window.location.pathname);
  });
  
  // Timeout fallback
  setTimeout(() => { netlifyIdentity.close(); }, 5000);
} else {
  // Normal flow: init + check login state
  netlifyIdentity.init();
  netlifyIdentity.on('init', user => {
    if (!user && !isLoginPage) window.location.href = '/login.html';
    if (user && isLoginPage) window.location.href = '/';
  });
}
```
